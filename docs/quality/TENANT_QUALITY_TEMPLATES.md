# قوالب الجودة الخاصة بالمستأجر — Tenant Quality Templates

> **النوع:** دستور — يصحّح فهماً جوهرياً حول ملكية قوالب الجودة.
> **الأهمية:** **حرِج.** سوء فهم هذه النقطة يكسر تعدّد المستأجرين أو يفرض هوية مدرسة على كل المدارس.
> **العلاقة:** يخدم [SIDRA_SYSTEM_DOCTRINE](../architecture/SIDRA_SYSTEM_DOCTRINE.md) §2 (التسمية الديناميكية) و[QUALITY_FORMS_AND_AUTOFILL_LAYER](./QUALITY_FORMS_AND_AUTOFILL_LAYER.md) §2.
> **آخر تحديث:** 2026-06-12.

---

## 1) التصحيح الحاسم

> **نماذج/أكواد «مدارس الفلاح الأهلية» الموجودة حالياً في الكود (مثل أكواد `QF-*`، الترويسات، أسماء النماذج) هي قوالب خاصة بمستأجر «الفلاح» — وليست افتراضات سِدرة العالمية.**

- طبقة «نماذج الجودة» **قدرة عالمية** في منصّة سِدرة.
- لكن **القالب · الكود · الترويسة · شعار المدرسة · لغة النموذج · الإتاحة (متى يظهر النموذج لمن)** كلها **لكل مدرسة / لكل مستأجر** (`school_id`-scoped).
- **مدرسة بلا برنامج جودة لا ترى نماذج الفلاح.** عرضها لها = تسريب هوية مستأجر + افتراض خاطئ بأن نظام جودة الفلاح هو نظام سِدرة العالمي.

---

## 2) ما هو عالمي وما هو خاص بالمستأجر

| العنصر | عالمي (سِدرة) | خاص بالمستأجر (لكل مدرسة) |
| --- | --- | --- |
| **القدرة** (وجود طبقة نماذج جودة) | ✅ | |
| **محرّك التعبئة + التصدير PDF** (الآلية) | ✅ | |
| **اسم المنتج المعروض** («سِدرة») | ✅ | |
| **قائمة القوالب المتاحة** | | ✅ |
| **أكواد النماذج** (مثل `QF-70-j-4-1`) | | ✅ |
| **ترويسة/شعار/اسم المدرسة في PDF** | | ✅ (من `schools.name`) |
| **تفعيل/تعطيل وحدات الجودة** | | ✅ |
| **لغة/صياغة بنود النموذج** | | ✅ |

## 3) الاستثناء المشروع (والمهم)

**مقبول تماماً** أن يَعرض قالب **خاص بمستأجر الفلاح** الاسم «مدارس الفلاح الأهلية» — **بشرط واحد:**

> أن يكون مُنمذَجاً صراحةً كـ **tenant-specific template** (مرتبط بـ `school_id` الخاص بالفلاح) — **لا** كنصّ مُثبَّت في قالب عام قابل لإعادة الاستخدام عبر كل المدارس.

الفرق الجوهري:

```
❌ خطأ (كسر مستأجر):  ثابت في مكوّن عام →  <Text>مدارس الفلاح الأهلية</Text>
✅ صحيح (ديناميكي):   <Text>{school.name}</Text>   // يقرأ اسم مستأجر المستخدم الحالي
✅ صحيح (قالب مستأجر): قالب مرتبط بـ school_id = «الفلاح» قد يحمل اسم الفلاح بحكم خصوصيته
```

## 4) الحالة الحالية (صدق) والفجوة

- **مؤكَّد:** `app/health/_components/HealthReports.tsx:103` يُثبّت «مدارس الفلاح الأهلية» في قالب PDF → **عيب تعدّد مستأجرين** (يجب تمرير `schoolName` ديناميكياً).
- **مُرجّح:** أنماط مماثلة في قوالب Activity / Secretary / Student-Affairs.
- **العلاج (المرحلة 3 — لا الآن):** تمرير `schoolName` ديناميكياً (app-code، بلا schema) + نمذجة القوالب الخاصة بمستأجر كـ tenant-specific صراحةً + إخفاء قوالب الفلاح عن المدارس التي لا تملك برنامجها.

---

## 5) قاعدة للمساهمين

1. **لا تُثبّت** اسم أي مدرسة في قالب/مكوّن عام. اقرأه من سياق المستأجر.
2. **لا تفترض** أن أكواد/نماذج الفلاح متاحة لكل المدارس.
3. عند بناء طبقة قوالب الجودة (المرحلة 3): اجعل **القالب كياناً مرتبطاً بـ `school_id`**، والإتاحة قراراً لكل مدرسة.
4. اقرأ [SIDRA_SYSTEM_DOCTRINE](../architecture/SIDRA_SYSTEM_DOCTRINE.md) §2 قبل لمس أي قالب أو ترويسة PDF.
